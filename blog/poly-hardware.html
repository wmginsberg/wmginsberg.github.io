Arduino, Rasberry Pi, Polymer, and Firebase

Posted by wmginsberg on may 28th, 2017 | 10 minute read

Did you ever want to work on home automation projects? Build your own smart security camera, or a robot, or a system to monitor your pet lizard's tank conditions including what you fed him and when? Me too! Particularly the last one. That's why I decided to combine my two hobbies, making cool hardware hacks, and making dumbinnovative web apps. Since the details of my projects aren't all that relevant, I figured I'd just share some things I learned about how oddly simple it is to stream data from hardware sensors to the web.

What tools did I use?

Ok, something that really frustrates me about any sort of hardware/software development is that any project takes a damn village to kick up. Here's an example: a stable monitoring system and pretty web app includes all of these products and services, but is actually pretty simple (and can be simpler):

    My version
  • Hardware: Raspberry Pi 3 Model B+ (running Raspbian), homemade 3.3V/5V Arduino board, DHT22 sensor
  • Cables & Power: 3 jumper cables, Micro USB 5V power supply, USB A to B cable
  • Web stuff: Polymer, Firebase, Nodejs, Heroku, various Python libraries (i.e. PySerial)
  • Languages: C/C++, Python, JS/HTML/CSS
    Simpler (and uglier!) version
  • Hardware: Raspberry Pi 3 Model B+ (running Raspbian), homemade 3.3V/5V Arduino board, DHT22 sensor
  • Cables & Power: 3 jumper cables, Micro USB 5V power supply, USB A to B cable
  • Web stuff: Flask, Heroku, PySerial, various Python libraries
  • Languages: C/C++, Python
